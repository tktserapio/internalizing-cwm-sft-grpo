## Negotiation Game Rules (Lewis et al. Style)

This is a simplified negotiation game inspired by Lewis et al.'s "Deal or No Deal" paper. Two players negotiate over how to divide a pool of items, where each player has private valuations.

### 1. Setup & Preliminaries
- **Players:** 2 (Player 0 and Player 1)
- **Items:** 3 item types: books (B), hats (H), balls (L)
- **Quantities:** 2 of each item type (6 items total)
- **Private Values:** Each player has private values for each item type (0-3 points each)
- **Goal:** Agree on a division that maximizes your points

### 2. Game Flow
1. **CHANCE node** assigns private valuations to each player
2. Players take turns making proposals or responding
3. A player can: **propose** a division, **accept** the current proposal, or **reject** and walk away
4. Game ends when a proposal is accepted or a player walks away

### 3. Actions
- **propose_X_Y_Z:** Propose that YOU get X books, Y hats, Z balls (opponent gets remainder)
- **accept:** Accept the current proposal on the table
- **reject:** Reject and end negotiation with no deal

### 4. Payoffs
- **If proposal accepted:** Each player gets points = sum(items received × item value)
- **If rejected (no deal):** Both players get 0 points
- Values are private: P0 doesn't know P1's values and vice versa

### 5. Move Notation
- **CHANCE action:** "v0_A_B_C_v1_D_E_F" where A,B,C are P0's values for books,hats,balls and D,E,F are P1's values
- **propose_X_Y_Z:** Propose you take X books, Y hats, Z balls
- **accept:** Accept current proposal
- **reject:** End with no deal

### 6. Strategic Notes
- Optimal negotiation requires inferring opponent's values from their proposals
- Zero-sum thinking fails: creative deals can create value for both
- Deadlines and walk-away threats affect bargaining power

### Example Trajectory
This shows a simple negotiation where P0 proposes and P1 accepts.

Step 0 (Chance/Setup): The game begins with a shared pool of resources to divide: 2 books, 2 hats, and 2 balls. The system randomly and secretly assigns private point valuations for these items to each player. Player 0 is assigned valuations of 3 points per book, 1 point per hat, and 0 points per ball. Player 1 is assigned valuations of 0 points per book, 2 points per hat, and 2 points per ball. Each player can only see their own private multipliers; the opponent's values remain strictly hidden.
Step 1 (Player 0): It is Player 0's turn to act first. From the available valid splits of the resource pool, Player 0 chooses to make a proposal. Player 0 proposes a split where they take 2 books, 0 hats, and 0 balls for themselves (leaving 0 books, 2 hats, and 2 balls for Player 1).
Step 2 (Player 1): It is Player 1's turn. Reviewing Player 0's proposed split, Player 1 must now decide whether to accept the deal, reject it entirely (often resulting in 0 points for both), or make a counter-proposal. Player 1 chooses to accept the deal.
Step 3 (Terminal): Because Player 1 accepted the proposal, an agreement is reached and the game ends immediately. Both players calculate their final payoffs based on their private valuations of the items they received. Player 0 received 2 books (2 items × 3 points), yielding a final payoff of +6.0. Player 1 received the remainder of the pool—2 hats (2 items × 2 points) and 2 balls (2 items × 2 points)—yielding a final payoff of +8.0. No further actions can be taken.

## Action String Specification
Player actions:
  - Propose: format "propose_<books>_<hats>_<balls>" (quantities for proposing player)
    e.g. "propose_2_1_0"
  - "accept": accept the current proposal
  - "reject": reject the current proposal
Chance actions: format "v0_<b>_<h>_<ba>_v1_<b>_<h>_<ba>" encoding valuations for both players
  e.g. "v0_3_1_0_v1_0_2_2"
