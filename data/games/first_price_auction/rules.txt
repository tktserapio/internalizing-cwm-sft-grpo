## First-Price Sealed-Bid Auction Rules

A first-price sealed-bid auction is a type of auction where bidders simultaneously submit sealed bids, and the highest bidder wins and pays their bid amount.

### 1. Setup & Preliminaries
- **Players:** 2 (Player 0 and Player 1)
- **Item Value:** Each player has a private valuation for the item, drawn uniformly from {1, 2, 3, 4, 5}
- **Bid Range:** Players can bid integer amounts from 0 to 5
- **Simultaneous Bidding:** Both players submit bids without seeing the other's bid

### 2. Game Flow
1. **CHANCE node** assigns private valuations to each player (dealt as "V0V1" e.g., "35" means P0 values at 3, P1 at 5)
2. **Player 0** secretly submits a bid (0-5)
3. **Player 1** secretly submits a bid (0-5) without seeing P0's bid
4. **Resolution:** Highest bidder wins the item

### 3. Winning and Payoffs
- The player with the highest bid wins the item
- **Winner's payoff:** (private valuation) - (bid amount)
- **Loser's payoff:** 0
- **Tie-breaker:** If bids are equal, Player 0 wins (arbitrary but deterministic)

### 4. Strategic Notes
- Bidding your true valuation guarantees zero profit if you win
- Optimal strategy involves shading your bid below your valuation
- The game has a Bayesian Nash equilibrium
- Overbidding (bid > valuation) risks negative payoff

### 5. Move Notation
- **CHANCE action:** "VxVy" where x is P0's valuation, y is P1's valuation (e.g., "35")
- **Bid action:** "bid_N" where N is the bid amount (e.g., "bid_3")

### Example Trajectory
This shows the expected game state at each step.

Step 0 (CHANCE): current_player=-1, legal_actions=["11", "12", ..., "55"], rewards=[0.0, 0.0]
  -> Action: "35"  # P0 values item at 3, P1 values at 5
Step 1: current_player=0, legal_actions=["bid_0", "bid_1", "bid_2", "bid_3", "bid_4", "bid_5"], rewards=[0.0, 0.0]
  -> Action: "bid_2"  # P0 bids 2 (shading below valuation of 3)
Step 2: current_player=1, legal_actions=["bid_0", "bid_1", "bid_2", "bid_3", "bid_4", "bid_5"], rewards=[0.0, 0.0]
  -> Action: "bid_4"  # P1 bids 4 (shading below valuation of 5)
Step 3 (TERMINAL): current_player=-4, legal_actions=[], rewards=[0.0, 1.0]
  # P1 wins with bid 4, pays 4, values at 5. Profit = 5 - 4 = 1
